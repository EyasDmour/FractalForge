1. Project Name
Fractal Forge

2. Project Type
Interactive mathematical learning website with visual simulations and a challenge-based
learning mode.

3. Project Goal
The goal of Fractal Forge is to help learners understand fractals through explanation,
visualization, and interaction.
Fractals can be difficult to understand when they are only explained using definitions and
formulas. Many important ideas, such as iteration, self-similarity, fractal dimension, complex
numbers, and formula-based behavior, become clearer when they are seen visually.
Fractal Forge transforms these ideas into an interactive website where users can explore
how simple mathematical rules create complex structures.
The project focuses on showing that fractals are not just beautiful images. They are
mathematical systems created through repeated rules.

4. Math Challenge Addressed
The main mathematical challenge addressed by this project is:
How can abstract fractal concepts be made easier to understand through interactive
visualization?

5. Project Description
Fractal Forge is an educational website that explains and visualizes fractals step by step.
The website begins with simple geometric fractals, such as the Sierpinski triangle and Koch
snowflake. These examples introduce the idea of repeated rules and self-similarity.
Then the project moves into fractal dimension, explaining why some fractals are more
complex than a line but do not fully fill a plane.
After that, the website introduces complex numbers and the complex plane. This prepares
the learner for formula-based fractals such as Julia sets, the Mandelbrot set, and Newton’s
fractal.
Each section of the website includes a short explanation and an interactive visual tool.
The project also includes a challenge mode where learners apply the ideas they studied.
The challenge mode is not the whole project; it is an active learning component that supports
the educational goal.

6. Mathematical Content Included
The project includes the following mathematical topics:

6.1 Fractals and Iteration
A fractal is a structure generated by repeatedly applying a simple rule.
The project introduces this idea using the Sierpinski triangle, where a repeated geometric
rule creates a detailed self-similar structure.

6.2 Geometric Fractals
Geometric fractals are created by applying repeated construction rules directly to shapes.
Examples used in the project:
●​ Sierpinski triangle
●​ Koch snowflake

6.3 Koch Snowflake

The Koch snowflake is used to show how a simple repeated rule can create an unusual
mathematical result.
At every iteration, each side is divided into three parts, the middle part is removed, and it is
replaced with two sides of a small outward equilateral triangle.
The number of sides increases according to:
Nn = 3 × 4^n
The side length decreases according to:
ln = s / 3^n
The perimeter is:
Pn = 3s(4/3)^n
As n increases, the perimeter approaches infinity, while the area remains finite.
This connects to the coastline paradox.

6.4 Fractal Dimension
The project explains that fractals can have non-integer dimensions.
The self-similarity dimension formula is:
D = log(N) / log(1/r)
For the Koch curve:
N=4
r = 1/3
So:
D = log(4) / log(3)
D ≈ 1.262
This means the Koch curve is more complex than a line but does not fully fill a
two-dimensional area.

6.5 Complex Numbers and the Complex Plane
The project introduces complex numbers in the form:
z = a + bi

Each complex number can be represented as a point on the complex plane.
This allows formula-based fractals to be visualized, because each pixel on the screen can
represent a complex number.

6.6 Julia Set
The Julia set is generated using:
z(n+1) = z(n)^2 + c
In the Julia set, c is fixed and different starting values of z are tested.
The final image is created by coloring points based on whether they stay bounded or
escape.

6.7 Mandelbrot Set
The Mandelbrot set uses the same formula:
z(n+1) = z(n)^2 + c
However, in the Mandelbrot set:
z0 = 0
and c changes.
Each point represents a value of c.
The Mandelbrot set shows which values of c create bounded behavior.

6.8 Julia and Mandelbrot Relationship
The project shows that the Mandelbrot set can be understood as a map of Julia sets.
A point selected from the Mandelbrot set can be used as the value of c to generate a
corresponding Julia set.
This is one of the main interactive features of the project.

6.9 Newton’s Fractal
Newton’s fractal is based on Newton’s method:
z(n+1) = z(n) - f(z(n)) / f′(z(n))
The project shows how applying Newton’s method across the complex plane creates colored
regions based on which root each point approaches.

This connects algebra, calculus, complex numbers, iteration, and visualization.

Working Prototype Draft
1. Prototype Overview
The working prototype of Fractal Forge is an interactive website that demonstrates the main
mathematical ideas of the project.
The prototype is designed to show the learning path of the project, from basic geometric
fractals to formula-based fractals.
The prototype includes interactive visual tools that allow the user to change values and
observe the mathematical results.

2. Prototype Features
Feature 1: Sierpinski Triangle Animation
The prototype includes a Sierpinski triangle animation.
The user can increase the number of iterations.
At each step, the triangle becomes more detailed.
Purpose:
This feature explains how a simple repeated rule creates a fractal.

Feature 2: Koch Snowflake Slider
The prototype includes a Koch snowflake visualizer.
The user can control the iteration level using a slider.
The website displays:
●​
●​
●​
●​

Number of sides
Side length
Perimeter behavior
Area behavior

Purpose:
This feature explains the idea of infinite perimeter and finite area.

Feature 3: Fractal Dimension Tool
The prototype includes a simple fractal dimension comparison.
The user can compare:
●​ Line: dimension 1
●​ Square: dimension 2
●​ Koch curve: dimension approximately 1.262
Purpose:
This feature explains that fractals can have non-integer dimensions.

Feature 4: Complex Plane Explorer
The prototype includes a complex plane interaction.
When the user moves the cursor over the plane, the website displays the complex number at
that location.
Example:
z = 0.4 + 0.2i
Purpose:
This feature prepares the user for formula-based fractals.

Feature 5: Julia Set Generator
The prototype includes a Julia set generator.
The user can change the real and imaginary parts of c.
When c changes, the Julia set changes.
Purpose:
This feature shows how a fixed parameter controls the appearance of a Julia set.

Feature 6: Mandelbrot Explorer
The prototype includes a Mandelbrot set viewer.
The user can explore the Mandelbrot set and observe its detailed boundary.
Purpose:
This feature shows how changing c creates a map of bounded and escaping behavior.

Feature 7: Mandelbrot-to-Julia Generator
The prototype includes a connection between the Mandelbrot set and Julia sets.
The screen is divided into two parts:
Left side: Mandelbrot set
Right side: Julia set
When the user clicks a point on the Mandelbrot set, that point becomes the value of c, and
the corresponding Julia set is generated.
Purpose:
This is the strongest prototype feature because it shows the relationship between the
Mandelbrot set and Julia sets directly.

Feature 8: Newton Fractal Visualization
The prototype includes a Newton fractal visualizer.
The user can see colored regions representing different roots.
The user can click a starting point and observe which root it approaches.
Purpose:
This feature explains how Newton’s method can create fractal boundaries.

Feature 9: Challenge Mode
The prototype includes a challenge mode.

In this mode, users answer or solve small interactive tasks based on the concepts they
learned.
Examples:
●​
●​
●​
●​
●​

Choose the correct rule for the Koch snowflake
Classify a fractal as geometric or formula-based
Adjust c to change a Julia set
Click a Mandelbrot point to generate a Julia set
Choose a starting point in Newton’s fractal that approaches a specific root

Purpose:
This mode supports active learning by making users apply the concepts.

3. Prototype User Flow
The prototype follows this order:
1.​ Introduction to fractals
2.​ Sierpinski triangle interaction
3.​ Geometric vs formula-based comparison
4.​ Koch snowflake visualizer
5.​ Fractal dimension tool
6.​ Complex plane explorer
7.​ Julia set generator
8.​ Mandelbrot explorer
9.​ Mandelbrot-to-Julia generator
10.​Newton fractal visualizer
11.​Challenge mode
This order allows the learner to move from simple visual ideas to more advanced
formula-based fractals.

The prototype is designed to make abstract mathematical ideas visible.
Instead of only reading formulas, the learner can interact with them.
The prototype allows learners to:
